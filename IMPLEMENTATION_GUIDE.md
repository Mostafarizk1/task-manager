# نظام إدارة المهام - دليل التنفيذ

## نظرة عامة

نظام شامل لإدارة المهام مع التحكم في الصلاحيات (RBAC)، دعم العملات المتعددة، إدارة المصاريف النثرية، والتقارير الشهرية المفصلة.

## الميزات الرئيسية

### 1. نظام الصلاحيات (Role-Based Access Control)

#### المدير (ADMIN)
- **الصلاحيات الكاملة:**
  - عرض جميع المهام من جميع المستخدمين
  - إضافة، تعديل، وحذف أي مهمة
  - الوصول الكامل للمعلومات المالية (السعر، الأجر، الربح)
  - إدارة المصاريف النثرية
  - عرض التقارير الشهرية المفصلة
  - إدارة المستخدمين

#### الموظف (USER)
- **الصلاحيات المحدودة:**
  - عرض وإدارة مهامه الشخصية فقط
  - إضافة وتعديل مهامه الخاصة
  - تغيير حالة مهامه (نشط، مراجعة، مكتمل)
  - **ممنوع تماماً من:**
    - رؤية أي مبالغ مالية (السعر، الأجر، الربح)
    - الوصول لقسم المصاريف النثرية
    - عرض التقارير المالية

### 2. إدارة المهام والعملات المتعددة

#### حقول المهمة
- **اسم المهمة**: عنوان وصفي للمهمة
- **اسم العميل**: اسم العميل المرتبط بالمهمة
- **موعد التسليم**: التاريخ المحدد لإنجاز المهمة
- **السعر الإجمالي**: المبلغ الكامل للمهمة (ADMIN فقط)
- **اسم المتعاون**: اسم المتعاون في المهمة (اختياري)
- **أجر المتعاون**: المبلغ المدفوع للمتعاون (ADMIN فقط)
- **العملة**: العملة المستخدمة (SAR, USD, EGP)
- **الحالة**: حالة المهمة (نشط، مراجعة، مكتمل)

#### العملات المدعومة
```typescript
enum Currency {
  SAR  // ريال سعودي 🇸🇦
  USD  // دولار أمريكي 🇺🇸
  EGP  // جنيه مصري 🇪🇬
}
```

#### حالات المهمة
```typescript
enum TaskStatus {
  ACTIVE     // نشط - المهمة قيد العمل
  REVIEW     // مراجعة - المهمة قيد المراجعة
  COMPLETED  // مكتمل - المهمة منتهية
}
```

#### الحسابات التلقائية
```
صافي الربح = السعر الإجمالي - أجر المتعاون
```
يتم حساب صافي الربح تلقائياً ويُعرض بنفس عملة المهمة.

### 3. قسم المصاريف النثرية (Petty Cash)

**الوصول:** ADMIN فقط

#### أنواع المصاريف
- نت (Internet)
- سلف (Advances)
- فواتير (Bills)
- اشتراكات (Subscriptions)
- أخرى (Other)

#### حقول المصروف
- **نوع المصروف**: تصنيف المصروف
- **المبلغ**: قيمة المصروف
- **العملة**: العملة المستخدمة (SAR, USD, EGP)
- **التاريخ**: تاريخ المصروف
- **ملاحظات**: وصف إضافي (اختياري)

### 4. لوحة التقارير الشهرية الذكية

**الوصول:** ADMIN فقط

#### فلتر الشهر والسنة
- اختيار الشهر المطلوب
- اختيار السنة المطلوبة
- الافتراضي: الشهر الحالي

#### إحصائيات مقسمة حسب العملة

لكل عملة (SAR, USD, EGP)، يتم عرض:

**1. إجمالي الدخل**
- مجموع السعر الإجمالي لجميع المهام المكتملة
- عدد المهام المكتملة

**2. صافي ربح المهام**
- مجموع أرباح المهام بعد خصم أجور المتعاونين
- الحساب: مجموع (السعر الإجمالي - أجر المتعاون)

**3. المصاريف النثرية**
- مجموع جميع المصاريف النثرية
- عدد المصاريف

**4. صافي الربح النهائي**
```
صافي الربح النهائي = صافي ربح المهام - المصاريف النثرية
```

#### مثال توضيحي (بالريال السعودي):
```
إجمالي الدخل: 10,000 ر.س (من 5 مهام)
صافي ربح المهام: 7,000 ر.س (بعد خصم أجور المتعاونين)
المصاريف النثرية: 1,500 ر.س (من 8 مصاريف)
─────────────────────────────────
صافي الربح النهائي: 5,500 ر.س ✓
```

## البنية التقنية

### قاعدة البيانات (Prisma Schema)

```prisma
enum Role {
  ADMIN
  USER
}

enum Currency {
  SAR
  USD
  EGP
}

enum TaskStatus {
  ACTIVE
  REVIEW
  COMPLETED
}

model User {
  id        String    @id @default(cuid())
  username  String    @unique
  password  String
  role      Role      @default(USER)
  tasks     Task[]
  expenses  Expense[]
}

model Task {
  id               String     @id @default(cuid())
  title            String
  clientName       String
  deadline         DateTime
  totalPrice       Float
  currency         Currency   @default(USD)
  collaboratorName String?
  collaboratorCut  Float      @default(0)
  netProfit        Float
  status           TaskStatus @default(ACTIVE)
  userId           String
  user             User       @relation(...)
}

model Expense {
  id          String   @id @default(cuid())
  type        String
  amount      Float
  currency    Currency @default(USD)
  description String?
  date        DateTime @default(now())
  userId      String
  user        User     @relation(...)
}
```

### API Routes

#### Tasks API (`/api/tasks`)
- `GET` - جلب المهام (ADMIN: جميع المهام، USER: مهامه فقط)
- `POST` - إنشاء مهمة جديدة
- `PUT /api/tasks/[id]` - تحديث مهمة
- `DELETE /api/tasks/[id]` - حذف مهمة

#### Expenses API (`/api/expenses`) - ADMIN فقط
- `GET` - جلب جميع المصاريف
- `POST` - إنشاء مصروف جديد
- `PUT /api/expenses/[id]` - تحديث مصروف
- `DELETE /api/expenses/[id]` - حذف مصروف

#### Reports API (`/api/reports/monthly`) - ADMIN فقط
- `GET?month=X&year=Y` - جلب التقرير الشهري

### الصفحات

#### للجميع
- `/login` - تسجيل الدخول
- `/dashboard` - لوحة التحكم الرئيسية

#### ADMIN فقط
- `/admin/panel` - إدارة المستخدمين
- `/admin/expenses` - إدارة المصاريف النثرية
- `/admin/reports` - التقارير الشهرية

## أمثلة الاستخدام

### سيناريو 1: إضافة مهمة جديدة (ADMIN)
1. تسجيل الدخول كمدير
2. الضغط على "+ مهمة جديدة"
3. ملء البيانات:
   - اسم المهمة: "تصميم موقع إلكتروني"
   - اسم العميل: "شركة ABC"
   - موعد التسليم: 2024-04-01
   - السعر الإجمالي: 5000
   - العملة: SAR
   - اسم المتعاون: "أحمد محمد"
   - أجر المتعاون: 2000
   - الحالة: نشط
4. صافي الربح يُحسب تلقائياً: 3000 ر.س

### سيناريو 2: إضافة مصروف نثري (ADMIN)
1. الذهاب إلى "المصاريف النثرية"
2. الضغط على "+ مصروف جديد"
3. ملء البيانات:
   - نوع المصروف: نت
   - المبلغ: 500
   - العملة: SAR
   - التاريخ: 2024-03-15
   - ملاحظات: "فاتورة الإنترنت - مارس"

### سيناريو 3: عرض التقرير الشهري (ADMIN)
1. الذهاب إلى "التقارير الشهرية"
2. اختيار الشهر والسنة
3. عرض الإحصائيات المفصلة لكل عملة:
   - الريال السعودي: الدخل، الأرباح، المصاريف، الصافي
   - الدولار الأمريكي: الدخل، الأرباح، المصاريف، الصافي
   - الجنيه المصري: الدخل، الأرباح، المصاريف، الصافي

### سيناريو 4: إدارة المهام (USER)
1. تسجيل الدخول كموظف
2. عرض المهام الشخصية فقط
3. إضافة مهمة جديدة (بدون معلومات مالية)
4. تغيير حالة المهمة من "نشط" إلى "مراجعة" أو "مكتمل"
5. تعديل المهام الخاصة فقط

## الأمان والصلاحيات

### التحقق من الصلاحيات
- جميع API routes تتحقق من تسجيل الدخول
- API routes الخاصة بالمصاريف والتقارير تتحقق من دور ADMIN
- USER يمكنه فقط الوصول لمهامه الخاصة
- المعلومات المالية مخفية تماماً عن USER في الواجهة

### تشفير كلمات المرور
- استخدام bcrypt لتشفير كلمات المرور
- Salt rounds: 10

### المصادقة
- NextAuth.js مع JWT
- Session-based authentication

## الملاحظات المهمة

1. **العملات منفصلة تماماً**: لا يتم خلط العملات في الحسابات
2. **الحسابات التلقائية**: صافي الربح يُحسب تلقائياً
3. **الصلاحيات صارمة**: USER لا يمكنه رؤية أي معلومات مالية
4. **التقارير الشهرية**: تعتمد على تاريخ اكتمال المهمة (updatedAt)
5. **المصاريف النثرية**: منفصلة تماماً عن مصاريف المهام (أجور المتعاونين)
